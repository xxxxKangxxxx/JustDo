# Just Do — 앱 기획 문서 v0.1

> 기간을 가진 체크 가능한 항목들을 캘린더 뷰로 보여주는 통합 Task 관리 앱

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | Just Do |
| 플랫폼 | iOS (우선), 웹앱 (우선), Android (추후) |
| 핵심 컨셉 | 할일과 일정을 하나의 항목으로 통합 — 기간, 시간, 체크 완료를 자유롭게 조합 |
| 디자인 방향 | 미니멀, 깔끔, 다크모드 지원 |
| 참고 앱 | 네이버 캘린더 + 투두리스트의 장점 결합 |

---

## 2. 핵심 문제 정의

기존 캘린더/To-Do 앱의 불편함:

1. **할일과 일정의 경계가 애매함**
   - 예: 삼성전자 지원 기간 (4/1~4/21) → 일정이기도 하고 할일이기도 한데, 기존 앱에서는 두 번 등록하거나 하나만 선택해야 함
2. **To-Do 앱에서 기간 설정이 불가**
   - 마감일 하나만 지정 가능, 시작~끝 기간을 설정해서 캘린더에 표시할 수 없음

---

## 3. 항목 타입

### 3-1. Task (할일 + 일정 통합)

항목 하나가 아래 속성들을 자유롭게 조합 가능.

| 속성 | 필수 여부 | 설명 |
|------|-----------|------|
| 제목 | 필수 | - |
| 날짜 범위 | 선택 | 시작일 ~ 마감일 |
| 특정 시간 고정 | 선택 | 예: 오후 2시 치과 예약 |
| 체크 완료 | 항상 가능 | 완료 처리 |
| 우선순위 | 선택 | 높음 / 중간 / 낮음 |
| 카테고리 | 선택 | 나 / 외부 |
| 태그 | 선택 | 자유 입력, 복수 가능 (#취업, #건강 등) |
| 하위 할일 (서브태스크) | 선택 | Task 안에 세부 항목 |
| Task 연결 (Dependency) | 선택 | 이전 Task / 다음 Task 연결 |
| 메모 / 설명 | 선택 | 자유 텍스트 |
| 알림 | 선택 | 날짜/시간 기반 알림 |

#### Task 연결 (Dependency) 시각화
- **웹:** 연결된 Task 흐름을 간트차트 또는 플로우 형태로 시각화
- **앱:** 이전 Task / 다음 Task 텍스트로 표시 (간단하게)
- 예시 흐름: `자소서 작성 → 삼성전자 지원 → 결과 확인`

---

### 3-2. Habit (반복 습관)

Task와 별도 카테고리로 분리 관리.

| 속성 | 필수 여부 | 설명 |
|------|-----------|------|
| 제목 | 필수 | - |
| 반복 주기 | 필수 | 매일 / 매주 특정 요일 / 매월 |
| 체크 완료 | 항상 가능 | 일별 완료 처리 |
| 목표 설정 | 선택 | 예: 하루 30분 |
| 카테고리 | 선택 | 나 / 외부 |
| 태그 | 선택 | 자유 입력 |
| 스트릭 | 자동 | 연속 달성일 카운트 |
| 달성률 통계 | 자동 | 주간/월간 달성률 |
| 알림 | 선택 | 반복 알림 |

## 4. 카테고리 & 태그

| 구분 | 내용 |
|------|------|
| 카테고리 | **나** (개인 할일, 습관, 일정) / **외부** (회사, 지원, 약속 등) |
| 태그 | 자유 입력, 복수 가능, 캘린더 뷰 필터링에 활용 |

## 5. 주요 화면 (View)

### 5-1. 캘린더 뷰 (메인)
- 기본: 월간 뷰
- 전환 가능: 주간 뷰
- Task와 Habit 모두 캘린더에 표시
- 카테고리/태그별 색상 구분
- 기간이 있는 Task는 날짜 범위 바(bar)로 표시

### 5-2. Task 상세 뷰
- 속성 전체 확인 및 편집
- 서브태스크 목록
- 이전/다음 Task 연결 표시 (앱)
- Task 흐름 시각화 (웹 전용)

### 5-3. Habit 뷰
- 습관 목록 및 오늘 완료 현황
- 스트릭 및 달성률 통계

### 5-4. 웹 전용 — Task Dependency 시각화
- 연결된 Task들의 선후 관계를 간트차트 또는 플로우차트로 표현

## 10. 화면 설계 (iOS 기준)

### 10-1. 네비게이션 구조
```
하단 탭 바
├── 🗓 홈 (캘린더 뷰) — 기본 진입 화면
├── 📊 통계
└── ⚙️ 설정

우측 상단 ➕ 버튼 — Task / Habit 빠른 추가
```

---

### 10-2. 홈 화면 (캘린더 뷰)

**상단 영역**
- 년/월 표시 + 좌우 화살표로 월 이동
- 드롭다운으로 월간 / 주간 / 일간 전환

**캘린더 영역**
- 날짜 셀에 점(dot)으로 Task/Habit 존재 표시
- 기간 있는 Task는 색상 바(bar)로 날짜 범위 연결

**하단 영역 (날짜 선택 시 펼쳐짐)**
- 카테고리별 그룹핑된 Task + Habit 목록
```
[나]
   ○  삼성전자 지원        4/1 ~ 4/21
   ○  독서 30분

[외부]
   ○  업체 미팅           오후 2시
   ○  팀 보고서 제출       4/21

[Habit]
   ○  운동 30분
   ○  독서
```

---

### 10-3. Task 추가 화면

**진입 방식**
- ➕ 버튼 → Bottom Sheet로 올라옴
- 위로 드래그 → 전체 화면으로 확장

**Bottom Sheet (기본)**
1. 제목 입력
2. Task / Habit 선택
3. 날짜 범위 설정
4. 시간 고정 여부
5. 카테고리 / 태그
6. 우선순위

**전체 화면 (상세 설정)**
- 서브태스크 추가
- 메모 / 설명
- 알림 설정
- 이전 / 다음 Task 연결

---

### 10-4. 통계 탭

| 섹션 | 내용 |
|------|------|
| **기간 전환** | 주간 / 월간 / 전체 |
| **Task** | 완료 수, 카테고리별 완료율, 미완료 현황 |
| **Habit** | 습관별 스트릭, 주간/월간 달성률 그래프, 전체 달성률 요약 |
| **월간 리포트** | 매월 자동 생성 — Task 완료율, Habit 달성률, 스트릭 하이라이트 |

---

### 10-5. 설정 탭

| 섹션 | 항목 |
|------|------|
| **계정** | 프로필, 로그아웃, 회원탈퇴 |
| **알림** | 알림 on/off, 알림 시간 설정 |
| **카테고리/태그** | 카테고리 색상 설정, 태그 관리 |
| **디스플레이** | 다크모드 전환, 캘린더 시작 요일 설정 |
| **구독** | 현재 플랜 확인, 업그레이드 |
| **데이터** | 데이터 내보내기, 캐시 삭제 |
| **앱 정보** | 버전, 이용약관, 개인정보처리방침 |

---

### 10-6. 웹 전용 화면

- Task Dependency 시각화 (간트차트 / 플로우차트)
- Task 추가/편집은 모달(Modal) 형태로 표시 (Bottom Sheet 대신)

## 11. 기술 스택

### 구조 개요
```
Just Do
├── iOS (Swift + SwiftUI)
│   ├── 앱
│   └── 위젯 (WidgetKit)
├── Web (Next.js)
│   └── 웹앱
└── Supabase (공통 백엔드)
    ├── Auth (Apple, Google 소셜 로그인)
    ├── PostgreSQL (DB)
    └── Realtime (iOS ↔ 웹 실시간 동기화)
```

### 스택 상세

| 영역 | 기술 | 이유 |
|------|------|------|
| **iOS 앱** | Swift + SwiftUI | 위젯(WidgetKit) 완벽 지원, 네이티브 성능 |
| **iOS 위젯** | WidgetKit | 소/중/대 위젯 구현 |
| **웹 앱** | Next.js + TypeScript | 웹 풀스택 경험 활용 |
| **스타일링 (웹)** | Tailwind CSS | Claude Code 궁합 최적 |
| **백엔드 / DB** | Supabase | Auth, 실시간 동기화, PostgreSQL 한 번에 해결 |
| **인증** | Supabase Auth | Apple / Google 소셜 로그인 내장 |
| **실시간 동기화** | Supabase Realtime | iOS ↔ 웹 자동 동기화 |

### Supabase 사용 시 주의사항
- **RLS (Row Level Security) 필수 설정** — 미설정 시 타 사용자 데이터 접근 가능
- `service_role key` 클라이언트 노출 금지, Next.js 환경변수로 관리
- 서버 리전: 일본 (ap-northeast-1) 권장 — 한국 리전 미제공
- 출시 시 Pro 플랜 고려 (무료 플랜은 2주 비활성화 시 일시 중지)
